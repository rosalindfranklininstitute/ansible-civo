.. Title

civo.cloud.firewall module -- Manage Civo firewalls and firewall rules
++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `civo.cloud collection <https://github.com/rosalindfranklininstitute/ansible-civo>`_ (version 0.0.3).

    It is not included in ``ansible-core``.
    To check whether it is installed, run :code:`ansible-galaxy collection list`.

    To install it, use: :code:`ansible\-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible\-civo.git`.

    To use it in a playbook, specify: :code:`civo.cloud.firewall`.

Synopsis
--------

.. Description

- Create or delete Civo firewalls.
- Declaratively manage firewall rules — missing rules are created, and with :emphasis:`purge\_rules=true` extra rules are removed.
- Rule matching uses the :emphasis:`label` field. Rules are compared on all attributes (direction, action, protocol, port, cidr); a rule whose label exists but whose attributes have changed is deleted and re\-created.
- Uses the :literal:`civo` CLI binary on the control node.

.. Examples

Examples
--------

.. code-block:: yaml+jinja

    - name: Create a firewall with SSH, HTTP, and HTTPS rules
      civo.cloud.firewall:
        region: LON1
        name: web-fw
        network: my-network
        rules:
          - label: allow-ssh
            direction: ingress
            protocol: tcp
            port: "22"
            cidr: "0.0.0.0/0"
          - label: allow-http
            direction: ingress
            protocol: tcp
            port: "80"
            cidr: "0.0.0.0/0"
          - label: allow-https
            direction: ingress
            protocol: tcp
            port: "443"
            cidr: "0.0.0.0/0"
        purge_rules: true

    - name: Delete a firewall
      civo.cloud.firewall:
        region: LON1
        name: web-fw
        state: absent
