.. Title

civo.cloud.instance module -- Manage Civo compute instances
+++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `civo.cloud collection <https://github.com/rosalindfranklininstitute/ansible-civo>`_ (version 0.0.3).

    It is not included in ``ansible-core``.
    To check whether it is installed, run :code:`ansible-galaxy collection list`.

    To install it, use: :code:`ansible\-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible\-civo.git`.

    To use it in a playbook, specify: :code:`civo.cloud.instance`.

.. version_added

.. rst-class:: ansible-version-added

New in civo.cloud 0.0.1

   :local:
   :depth: 1

Synopsis
--------

.. Description

- Create, update, start, stop, reboot, or delete Civo compute instances.
- Supports in\-place updates for :emphasis:`firewall`\ , :emphasis:`tags`\ , and :emphasis:`size` (resize) when an instance already exists.
- Uses the :literal:`civo` CLI binary on the control node.

.. Examples

Examples
--------

.. code-block:: yaml+jinja

    - name: Create an instance
      civo.cloud.instance:
        region: LON1
        hostname: web-01.example.com
        size: g4s.small
        diskimage: ubuntu-noble
        network: my-network
        firewall: web-fw
        ssh_key: my-key
        initial_user: ubuntu
        wait: true
      register: vm

    - name: Print public IP
      ansible.builtin.debug:
        msg: "Public IP: {{ vm.instance.public_ip }}"

    - name: Resize an instance
      civo.cloud.instance:
        region: LON1
        hostname: web-01.example.com
        size: g4s.medium

    - name: Stop an instance
      civo.cloud.instance:
        region: LON1
        hostname: web-01.example.com
        state: stopped

    - name: Gracefully reboot an instance (ACPI signal, equivalent to OS reboot)
      civo.cloud.instance:
        region: LON1
        hostname: web-01.example.com
        state: rebooted
        wait: true

    - name: Hard-reboot an instance (power cycle — use when soft reboot hangs)
      civo.cloud.instance:
        region: LON1
        hostname: web-01.example.com
        state: hard_rebooted
        wait: true

    - name: Delete an instance
      civo.cloud.instance:
        region: LON1
        hostname: web-01.example.com
        state: absent
