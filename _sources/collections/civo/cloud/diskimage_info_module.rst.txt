.. Title

civo.cloud.diskimage_info module -- List available Civo disk images (VM templates)
++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `civo.cloud collection <https://github.com/rosalindfranklininstitute/ansible-civo>`_ (version 0.0.3).

    It is not included in ``ansible-core``.
    To check whether it is installed, run :code:`ansible-galaxy collection list`.

    To install it, use: :code:`ansible\-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible\-civo.git`.

    To use it in a playbook, specify: :code:`civo.cloud.diskimage_info`.

New in civo.cloud 0.0.1

Synopsis
--------

.. Description

- Returns all disk images available in a Civo region, or filters to a single image by name or ID.
- Disk image names are the values accepted by the :literal:`diskimage` parameter of :ref:`civo.cloud.instance <ansible_collections.civo.cloud.instance_module>` (e.g. :literal:`debian\-11`\ , :literal:`ubuntu\-jammy`\ ).
- Uses the :literal:`civo` CLI binary on the control node.

.. Examples

Examples
--------

.. code-block:: yaml+jinja

    - name: List all available disk images
      civo.cloud.diskimage_info:
        region: LON1
      register: images

    - name: Show all image names
      ansible.builtin.debug:
        msg: "{{ images.diskimages | map(attribute='name') | list }}"

    - name: Look up a specific image by name
      civo.cloud.diskimage_info:
        region: LON1
        name: debian-11
      register: debian11

    - name: List only Ubuntu images
      civo.cloud.diskimage_info:
        region: LON1
        distribution: ubuntu
      register: ubuntu_images

    - name: Use the image ID when creating an instance
      civo.cloud.instance:
        region: LON1
        hostname: web-01
        diskimage: "{{ debian11.diskimages[0].name }}"
        size: g3.xsmall
        state: present
