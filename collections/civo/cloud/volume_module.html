civo.cloud.volume module – Manage Civo block storage volumes

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.volume.

New in civo.cloud 0.0.1

Synopsis

  • Create, resize, attach, detach, or delete Civo persistent volumes.

  • Idempotent attach/detach — checks current attachment state before acting.

  • Supports in-place resize when size_gb is increased on an existing volume.

  • Uses the civo CLI binary on the control node.

Parameters

Parameter

Comments

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

bootable

boolean

Whether this volume should be bootable.

Choices:

  • false ← (default)

  • true

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

instance

string

Hostname or ID of the instance to attach the volume to.

Required when state=attached.

name

string / required

Name of the volume.

network

string

Name or ID of the network to create the volume in.

Default: "default"

region

string

Civo region identifier.

Default: "LON1"

size_gb

integer

Size of the volume in gigabytes.

Can be increased on an existing volume (resize). Decreasing is not supported by the Civo API.

Default: 10

state

string

present ensures the volume exists (detached).

absent deletes the volume.

attached attaches the volume to instance.

detached detaches the volume from any instance.

Choices:

  • "present" ← (default)

  • "absent"

  • "attached"

  • "detached"

Examples

- name: Create a 50 GB volume
  civo.cloud.volume:
    region: LON1
    name: data-vol
    size_gb: 50

- name: Attach volume to an instance
  civo.cloud.volume:
    region: LON1
    name: data-vol
    instance: web-01.example.com
    state: attached

- name: Resize volume to 100 GB
  civo.cloud.volume:
    region: LON1
    name: data-vol
    size_gb: 100

- name: Detach volume
  civo.cloud.volume:
    region: LON1
    name: data-vol
    state: detached

- name: Delete a volume
  civo.cloud.volume:
    region: LON1
    name: data-vol
    state: absent

Return Values

Common return values are documented here, the following are the fields unique to this module:

Key

Description

volume

dictionary

Details of the managed volume.

Returned: when state is present, attached, or detached

bootable

boolean

Whether the volume is marked as bootable.

Returned: success

Sample: false

id

string

Volume UUID.

Returned: success

Sample: "a1b2c3d4-0000-0000-0000-000000000000"

instance_id

string

Hostname (not UUID) of the instance the volume is attached to. Empty string when the volume is not attached. Note: the Civo CLI stores the instance hostname here, not its UUID.

Returned: success

Sample: "web-01.example.com"

name

string

Volume name.

Returned: success

Sample: "data-vol"

network_id

string

Name or UUID of the network the volume belongs to. Returns the network name when the default network is used, or the UUID for custom networks.

Returned: success

Sample: "default"

size_gigabytes

string

Volume size as returned by the Civo CLI, including a unit suffix (e.g. "10 GB"). Strip the suffix before numeric comparison.

Returned: success

Sample: "10 GB"

status

string

Volume status (available, creating, resizing, etc.).

Returned: success

Sample: "available"
