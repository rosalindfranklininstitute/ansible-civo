civo.cloud.kubernetes module – Manage Civo Kubernetes clusters

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.kubernetes.

New in civo.cloud 0.0.1

Synopsis

  • Create, scale, upgrade, or delete Civo Kubernetes (k3s) clusters.

  • Supports in-place node-count scaling when the cluster already exists.

  • When the cluster has a single pool (the default), node_count targets that pool. When multiple pools exist you must supply pool_id to identify which pool to scale; otherwise the module fails to avoid silently operating on the wrong pool.

  • Supports in-place Kubernetes version upgrade via upgrade_version.

  • Returns the kubeconfig for an active cluster.

  • Uses the civo CLI binary on the control node.

Parameters

Parameter

Comments

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

applications

list / elements=string

List of Civo Marketplace application names to pre-install.

Default: []

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

cni

string

Container Network Interface plugin.

Choices:

  • "flannel" ← (default)

  • "cilium"

firewall

string

Name or ID of the firewall.

name

string / required

Name of the Kubernetes cluster.

network

string

Name or ID of the network.

Default: "default"

node_count

integer

Desired number of worker nodes in the target pool.

When the cluster has only one pool this targets that pool.

When the cluster has multiple pools pool_id must also be supplied.

When changed on an existing cluster the target pool is scaled in place.

Default: 3

node_size

string

Node pool instance size slug.

Default: "g4s.kube.medium"

pool_id

string

ID of the node pool to scale.

Required when the cluster has more than one pool and node_count is being changed.

Ignored during cluster creation (the first pool is always created automatically).

region

string

Civo region identifier.

Default: "LON1"

state

string

Whether the cluster should exist.

Choices:

  • "present" ← (default)

  • "absent"

timeout

integer

Seconds to wait for the cluster to become ready.

Default: 600

upgrade_version

string

Target Kubernetes version string to upgrade an existing cluster to (e.g. 1.29.2-k3s1).

Ignored when creating a new cluster; use version for that.

Calls civo kubernetes upgrade CLUSTER UPGRADE_VERSION and waits for the cluster to return to ACTIVE status.

version

string

Kubernetes version string (e.g. 1.28.7-k3s1). Empty means latest.

Default: ""

wait

boolean

Wait for the cluster to become ready before returning.

Choices:

  • false

  • true ← (default)

Examples

- name: Create a 3-node Kubernetes cluster
  civo.cloud.kubernetes:
    region: LON1
    name: my-cluster
    node_size: g4s.kube.medium
    node_count: 3
    network: my-network
    cni: flannel
    wait: true
  register: k8s

- name: Save kubeconfig locally
  ansible.builtin.copy:
    content: "{{ k8s.cluster.kubeconfig }}"
    dest: ~/.kube/my-cluster.yaml
    mode: "0600"

- name: Scale the default (only) pool to 5 nodes
  civo.cloud.kubernetes:
    region: LON1
    name: my-cluster
    node_count: 5

- name: Scale a specific pool when the cluster has multiple pools
  civo.cloud.kubernetes:
    region: LON1
    name: my-cluster
    node_count: 2
    pool_id: "aaaa-bbbb-cccc"

- name: Upgrade the cluster to a newer Kubernetes version
  civo.cloud.kubernetes:
    region: LON1
    name: my-cluster
    upgrade_version: "1.29.2-k3s1"
    wait: true

- name: Delete a cluster
  civo.cloud.kubernetes:
    region: LON1
    name: my-cluster
    state: absent
