civo.cloud.quota_info module – Return current Civo account quota and usage

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.quota_info.

New in civo.cloud 0.0.1

  • Synopsis

  • Parameters

  • Examples

  • Return Values

Synopsis

  • Returns the quota limits and current usage for a Civo account in a given region.

  • Useful for capacity-planning checks before provisioning resources.

  • Uses the civo CLI binary on the control node.

Parameters

Parameter

Comments

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

region

string

Civo region identifier.

Default: "LON1"

Examples

- name: Get account quota and usage
  civo.cloud.quota_info:
    region: LON1
  register: quota

- name: Show instance usage vs limit
  ansible.builtin.debug:
    msg: >-
      Instances: {{ quota.quota.instance_count_usage }}
      / {{ quota.quota.instance_count_limit }}

- name: Fail if near instance limit
  ansible.builtin.fail:
    msg: "Instance quota nearly exhausted!"
  when: >-
    quota.quota.instance_count_usage | int
    >= (quota.quota.instance_count_limit | int * 0.9) | int
