civo.cloud.objectstore module – Manage Civo S3-compatible object stores

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.objectstore.

New in civo.cloud 0.0.1

Synopsis

  • Create or delete Civo object store buckets.

  • When a store is created without an explicit owner credential, Civo automatically generates a dedicated credential for it. Those auto-created credentials are not removed when the store is deleted. Set purge_credentials=true on the delete call to clean them up.

  • Uses the civo CLI binary on the control node.

Parameters

Parameter

Comments

access_key_id

string

Access key ID of existing object store credentials to associate.

When omitted, new credentials are generated automatically.

Maps to the --owner-access-key CLI flag.

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

max_size_gb

integer

Maximum size of the object store in gigabytes.

0 means unlimited.

name

string / required

Name of the object store.

purge_credentials

boolean

When true and state=absent, also delete the object store’s owner credential after the store is removed.

The module looks up the linked credential via civo objectstore show (which returns the accesskey field), then cross-references civo objectstore credential ls to find its name and deletes it.

Only set this when you know the credential is not shared with other stores. If you created the store with an explicit access_key_id pointing to a shared credential, leave this as false.

Has no effect when state=present.

Choices:

  • false ← (default)

  • true

region

string

Civo region identifier.

Default: "LON1"

state

string

Whether the object store should exist.

Choices:

  • "present" ← (default)

  • "absent"

timeout

integer

Seconds to wait for the object store to become ready.

Default: 300

wait

boolean

Wait for the object store to become ready before returning.

Choices:

  • false

  • true ← (default)

Examples

- name: Create an object store with a 500 GB limit
  civo.cloud.objectstore:
    region: LON1
    name: my-bucket
    max_size_gb: 500
    wait: true
  register: bucket

- name: Print S3 endpoint
  ansible.builtin.debug:
    msg: "Endpoint: {{ bucket.objectstore.endpoint }}"

- name: Delete an object store and its auto-created credential
  civo.cloud.objectstore:
    region: LON1
    name: my-bucket
    state: absent
    purge_credentials: true
