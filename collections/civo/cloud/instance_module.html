civo.cloud.instance module – Manage Civo compute instances

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.instance.

New in civo.cloud 0.0.1

  • Parameters

  • See Also

Synopsis

  • Create, update, start, stop, reboot, or delete Civo compute instances.

  • Supports in-place updates for firewall, tags, and size (resize) when an instance already exists.

  • Uses the civo CLI binary on the control node.

Parameters

Parameter

Comments

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

diskimage

string

Disk image name or ID (e.g. ubuntu-noble).

Default: "ubuntu-noble"

firewall

string

Name or ID of the firewall.

When changed on an existing instance the firewall assignment is updated.

hostname

string / required

Hostname of the instance (used as the unique identifier).

initial_user

string

Username to create on the instance.

network

string

Name or ID of the network.

Default: "default"

public_ip

string

Whether to assign a public IP (create) or not (none).

Choices:

  • "create" ← (default)

  • "none"

region

string

Civo region identifier.

Default: "LON1"

script

string

User-data / cloud-init script to run on first boot.

size

string

Instance size slug (e.g. g4s.small, g4s.medium).

When changed on an existing instance the instance is resized in place.

Default: "g4s.small"

ssh_key

string

Name or ID of the SSH key to inject.

state

string

present ensures the instance exists and is running.

absent deletes the instance.

started ensures the instance is running (creates if needed).

stopped ensures the instance is stopped.

rebooted sends a graceful (ACPI) reboot signal. Always reports changed=True. Use wait to block until the instance returns to ACTIVE.

hard_rebooted performs a hard power-cycle reboot. Always reports changed=True. Use with care — data loss may occur if the instance OS does not flush write buffers before the power cut.

Choices:

  • "present" ← (default)

  • "absent"

  • "started"

  • "stopped"

  • "rebooted"

  • "hard_rebooted"

tags

string

Space-separated list of tags.

When changed on an existing instance the tags are updated in place.

timeout

integer

Seconds to wait for the instance to become active.

Default: 600

wait

boolean

Wait for the instance to become ACTIVE before returning.

Choices:

  • false

  • true ← (default)

Examples

- name: Create an instance
  civo.cloud.instance:
    region: LON1
    hostname: web-01.example.com
    size: g4s.small
    diskimage: ubuntu-noble
    network: my-network
    firewall: web-fw
    ssh_key: my-key
    initial_user: ubuntu
    wait: true
  register: vm

- name: Print public IP
  ansible.builtin.debug:
    msg: "Public IP: {{ vm.instance.public_ip }}"

- name: Resize an instance
  civo.cloud.instance:
    region: LON1
    hostname: web-01.example.com
    size: g4s.medium

- name: Stop an instance
  civo.cloud.instance:
    region: LON1
    hostname: web-01.example.com
    state: stopped

- name: Gracefully reboot an instance (ACPI signal, equivalent to OS reboot)
  civo.cloud.instance:
    region: LON1
    hostname: web-01.example.com
    state: rebooted
    wait: true

- name: Hard-reboot an instance (power cycle — use when soft reboot hangs)
  civo.cloud.instance:
    region: LON1
    hostname: web-01.example.com
    state: hard_rebooted
    wait: true

- name: Delete an instance
  civo.cloud.instance:
    region: LON1
    hostname: web-01.example.com
    state: absent
