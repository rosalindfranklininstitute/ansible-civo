civo.cloud.database module – Manage Civo managed databases

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.database.

New in civo.cloud 0.0.1

Synopsis

  • Create or delete Civo managed database clusters (MySQL or PostgreSQL).

  • Uses the civo CLI binary on the control node.

  • Note: --software is the CLI flag for the engine type and --version for the version string — the module maps the user-facing engine/version parameters to those flags.

Parameters

Parameter

Comments

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

engine

string

Database engine type.

Currently only postgresql is supported by Civo.

Run civo database versions to see available engines and versions in your region.

Choices:

  • "postgresql" ← (default)

firewall

string

Name or ID of the firewall.

name

string / required

Name of the database cluster.

network

string

Name or ID of the network.

Default: "default"

nodes

integer

Number of database nodes.

Default: 1

region

string

Civo region identifier.

Default: "LON1"

size

string

Database node size slug (e.g. g3.db.small).

Default: "g3.db.small"

state

string

Whether the database should exist.

Choices:

  • "present" ← (default)

  • "absent"

timeout

integer

Seconds to wait for the database to become ready.

Default: 600

version

string

Engine version string.

Run civo database versions to list available versions for your region.

Default: "17"

wait

boolean

Wait for the database to become ready before returning.

Choices:

  • false

  • true ← (default)

Examples

- name: Create a PostgreSQL database
  civo.cloud.database:
    region: LON1
    name: myapp-db
    engine: postgresql
    version: "17"
    size: g3.db.small
    nodes: 1
    network: my-network
    wait: true
  register: db

- name: Print database endpoint
  ansible.builtin.debug:
    msg: "DB endpoint: {{ db.database.endpoint }}"

- name: Delete a database
  civo.cloud.database:
    region: LON1
    name: myapp-db
    state: absent
