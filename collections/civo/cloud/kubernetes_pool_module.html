civo.cloud.kubernetes_pool module – Manage node pools in a Civo Kubernetes cluster

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.kubernetes_pool.

New in civo.cloud 0.0.1

Synopsis

  • Create, scale, or delete additional node pools in an existing Civo Kubernetes cluster.

  • state=present creates the pool if it does not exist, or scales it to node_count if it already exists.

  • state=absent deletes the pool.

  • The default pool created with the cluster is not managed by this module; use civo.cloud.kubernetes with node_count for that.

  • Uses the civo CLI binary on the control node.

Parameters

Parameter

Comments

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

cluster

string / required

Name of the Kubernetes cluster to manage pools for.

name

string

Name for the node pool. Used only during creation (--name flag). When omitted the Civo API generates a name automatically.

Default: ""

node_count

integer

Desired number of nodes in the pool.

Default: 1

node_size

string

Instance size slug for nodes in this pool.

Default: "g4s.kube.medium"

pool_id

string

ID of an existing pool to update or delete. When supplied the module locates the pool by ID rather than by name. Required for state=absent when more than one pool exists and the pool cannot be identified by name alone.

Default: ""

region

string

Civo region identifier.

Default: "LON1"

state

string

Whether the node pool should exist.

Choices:

  • "present" ← (default)

  • "absent"

timeout

integer

Seconds to wait for the pool to reach the desired node count.

Default: 600

wait

boolean

Wait for the pool to reach the desired node count after creation or scaling.

Choices:

  • false

  • true ← (default)

Examples

- name: Add a GPU node pool to an existing cluster
  civo.cloud.kubernetes_pool:
    region: LON1
    cluster: my-cluster
    name: gpu-pool
    node_size: g4g.kube.large
    node_count: 2
    wait: true
  register: pool

- name: Scale the pool to 4 nodes
  civo.cloud.kubernetes_pool:
    region: LON1
    cluster: my-cluster
    pool_id: "{{ pool.pool.id }}"
    node_count: 4

- name: Delete the pool
  civo.cloud.kubernetes_pool:
    region: LON1
    cluster: my-cluster
    pool_id: "{{ pool.pool.id }}"
    state: absent
