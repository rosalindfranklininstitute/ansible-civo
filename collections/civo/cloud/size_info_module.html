civo.cloud.size_info module – List available Civo instance / Kubernetes / database sizes

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.size_info.

New in civo.cloud 0.0.1

  • Parameters

Synopsis

  • Returns all sizes (plan types) available in a Civo region.

  • Optionally filters by resource type using the filter parameter.

  • The name field of each size is the value to pass to civo_instance, civo_kubernetes, or civo_database as the size parameter.

  • Uses the civo CLI binary on the control node.

Parameters

Parameter

Comments

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

region

string

Civo region identifier.

Default: "LON1"

resource_type

string

Filter sizes by resource type.

When omitted, all sizes are returned.

Choices:

  • "instance"

  • "kubernetes"

  • "database"

Examples

- name: List all sizes
  civo.cloud.size_info:
    region: LON1
  register: all_sizes

- name: List only instance sizes
  civo.cloud.size_info:
    region: LON1
    resource_type: instance
  register: instance_sizes

- name: Show instance size names
  ansible.builtin.debug:
    msg: "{{ instance_sizes.sizes | map(attribute='name') | list }}"

- name: List only Kubernetes node sizes
  civo.cloud.size_info:
    region: LON1
    resource_type: kubernetes
  register: k8s_sizes

- name: List database sizes
  civo.cloud.size_info:
    region: LON1
    resource_type: database
  register: db_sizes

Return Values

Common return values are documented here, the following are the fields unique to this module:

Key

Description

sizes

list / elements=dictionary

List of size dicts matching the query.

Returned: always

cpu_cores

string

Number of vCPU cores (string as returned by the CLI).

Returned: success

Sample: "1"

description

string

Human-readable size description.

Returned: success

Sample: "Extra Small"

disk_gb

string

Disk size in gigabytes (string as returned by the CLI).

Returned: success

Sample: "25"

name

string

Size slug — pass this to size parameters in resource modules.

Returned: success

Sample: "g3.xsmall"

ram_mb

string

RAM in megabytes (string as returned by the CLI).

Returned: success

Sample: "1024"

selectable

string

Whether this size can be selected when creating resources.

Returned: success

Sample: "true"

type

string

Resource type the size applies to (Instance, Kubernetes, Database).

Returned: success

Sample: "Instance"
