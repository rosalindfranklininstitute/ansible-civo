civo.cloud.kubernetes_node module – Recycle or delete a node in a Civo Kubernetes cluster

Note

This module is part of the civo.cloud collection (version 0.0.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install git+https://github.com/rosalindfranklininstitute/ansible-civo.git.

To use it in a playbook, specify: civo.cloud.kubernetes_node.

New in civo.cloud 0.0.1

Synopsis

  • Manages individual nodes (instances) within a Civo Kubernetes cluster.

  • state=recycle replaces the node with a fresh instance while keeping the pool size the same. Calls civo kubernetes recycle CLUSTER --node NODE.

  • state=absent permanently removes the node from its pool, reducing the pool size by one. Calls civo kubernetes node-pool instance-delete CLUSTER --instance-id ID --node-pool-id POOL_ID.

  • The node is identified by its hostname (e.g. k3s-my-cluster-abc12-1).

  • Uses the civo CLI binary on the control node.

Parameters

Parameter

Comments

api_key

string

Civo API token.

Falls back to the CIVO_TOKEN environment variable, then to the active key in ~/.civo.json (the civo CLI config), when not set.

civo_binary

string

Path to the civo CLI binary.

Default: "civo"

cluster

string / required

Name of the Kubernetes cluster that owns the node.

node

string / required

Hostname of the node to act on (e.g. k3s-my-cluster-abc12-1). Obtain available node names from civo.cloud.kubernetes_pool_info or civo kubernetes node-pool instance-ls CLUSTER.

pool_id

string

ID of the node pool that contains the node. Required for state=absent. Optional for state=recycle (the module will search all pools when omitted, which adds extra API calls).

region

string

Civo region identifier.

Default: "LON1"

state

string / required

recycle — replace the node with a new instance (same pool size).

absent — permanently delete the node instance from its pool.

Choices:

  • "recycle"

  • "absent"

Examples

- name: Recycle a node (replace with a fresh instance)
  civo.cloud.kubernetes_node:
    region: LON1
    cluster: my-cluster
    node: k3s-my-cluster-abc12-1
    state: recycle

- name: Delete a node from its pool (reduces pool size by 1)
  civo.cloud.kubernetes_node:
    region: LON1
    cluster: my-cluster
    node: k3s-my-cluster-abc12-1
    pool_id: "aaaa-bbbb-cccc"
    state: absent
